# Contribution Guidelines

This project is released with a [Contributor Code of Conduct](code-of-conduct.md). By participating, you agree to abide by its terms.

---

Please ensure your pull request adheres to the following:

- **One resource per line** - Add each link in the form: `- [Title](URL) - Short description.`
- **Put new entries in the right section** - Match the existing category (or suggest a new one in the PR description).
- **Use a stable, public URL** - Prefer official sites, docs, or canonical repos over temporary or paywalled links.
- **Keep descriptions concise** - One short sentence; no marketing fluff.
- **Check for duplicates** - Search the README before adding.
- **Alphabetize or keep logical order** - Within a section, maintain consistent ordering (e.g. alphabetical by title).

Thank you for contributing!

## Updating your PR

If maintainers request changes, you can edit your existing PR (no need to open a new one). [Here is a guide](https://github.com/RichardLitt/knowledge/blob/master/github/amending-a-commit-guide.md) on amending commits and updating a PR.
